# Инструкция по настройке

## Требования

- Proxmox с виртуальной машиной OpenMediaVault
- Docker и Docker Compose на OMV
- Браузер Chrome или Edge (для функции автосохранения файлов)

## 1. Развернуть на сервере

### Клонировать репозиторий локально

```bash
git clone https://github.com/Andreyhiitola/whisper-ui.git
cd whisper-ui
```

### Создать директории на сервере OMV

```bash
ssh root@192.168.1.73
mkdir -p /srv/dev-disk-by-uuid-e3906cb9-c585-4088-9de4-278d2769849e/whisper_ui/html
mkdir -p /srv/dev-disk-by-uuid-e3906cb9-c585-4088-9de4-278d2769849e/whisper_ui/conf
mkdir -p /srv/dev-disk-by-uuid-e3906cb9-c585-4088-9de4-278d2769849e/wisper_
```

### Скопировать файлы на сервер

```bash
scp html/index.html root@192.168.1.73:/srv/dev-disk-by-uuid-e3906cb9-c585-4088-9de4-278d2769849e/whisper_ui/html/index.html
scp nginx/default.conf root@192.168.1.73:/srv/dev-disk-by-uuid-e3906cb9-c585-4088-9de4-278d2769849e/whisper_ui/conf/default.conf
scp docker-compose.yml root@192.168.1.73:/srv/dev-disk-by-uuid-e3906cb9-c585-4088-9de4-278d2769849e/compose/whisper/whisper.yml
```

### Запустить контейнеры

```bash
ssh root@192.168.1.73
cd /srv/dev-disk-by-uuid-e3906cb9-c585-4088-9de4-278d2769849e/compose/whisper
docker compose -f whisper.yml up -d
```

## 2. Настроить Groq API

1. Зарегистрироваться на [console.groq.com](https://console.groq.com)
2. Создать API ключ: **API Keys → Create API Key**
3. Открыть интерфейс: `http://192.168.1.73:8090`
4. Перейти во вкладку **2. Анализ и чат**
5. Нажать **⚙️ Настройки Groq API**
6. Вставить ключ (начинается с `gsk_...`)
7. Выбрать модель: `llama-3.3-70b-versatile` (лучший русский)

API ключ сохраняется в браузере автоматически.

## 3. Рабочий процесс

### Транскрибация
1. Открыть `http://192.168.1.73:8090`
2. Вкладка **1. Транскрибация**
3. Загрузить аудиофайл(ы) с диктофона
4. Выбрать язык: Русский
5. Нажать **Транскрибировать все**
6. После завершения нажать **→ В анализ**

### Анализ
1. Вкладка **2. Анализ и чат**
2. Вставить или загрузить сценарий проверки (текст / PDF / скрин)
3. Выбрать шаблон анализа:
   - 📝 **Свободный отчёт** — полный анализ в свободной форме
   - 📊 **Таблица оценки** — выполнен / частично / не выполнен
   - 🎭 **Тональность** — оценка поведения и общения персонала
   - 🚨 **Критические нарушения** — только серьёзные отклонения
4. Нажать **Анализировать**
5. Уточнить детали в чате если нужно
6. Скопировать результат

## 4. Деплой обновлений

При изменении файлов запустить скрипт:

```bash
cd ~/Desktop/whisper-ui
./deploy.sh "описание изменений"
```

Скрипт автоматически коммитит, пушит на GitHub и копирует файлы на сервер.

## 6. Выбор модели Whisper

В `docker-compose.yml` параметр `ASR_MODEL`:

| Модель | Размер | Качество | Скорость |
|--------|--------|----------|----------|
| tiny   | 75 MB  | низкое   | быстро   |
| base   | 145 MB | базовое  | быстро   |
| small  | 460 MB | хорошее  | средне   |
| medium | 1.5 GB | высокое  | медленно |

Текущая модель: **small**
