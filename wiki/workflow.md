# Рабочий процесс проверки

## Схема

```
Диктофон → Whisper (транскрипция) → Groq или DeepSeek (анализ) → Отчёт
                                         ↑
                              Сценарий: текст / PDF / картинка
```

## Пошагово

### 1. До проверки
- Получить сценарий (текст / PDF / скриншот)

### 2. Проверка
- Записать разговор на диктофон

### 3. Транскрибация
- Открыть `http://192.168.1.73:8090`
- Вкладка **1. Транскрибация**
- Загрузить аудиофайл → **Транскрибировать**
- Нажать **→ В анализ**

### 4. Анализ
- Вкладка **2. Анализ и чат**
- Открыть **⚙️ Настройки API**, выбрать провайдер:
  - **Groq** — быстро, бесплатно
  - **DeepSeek** — качественнее, ~$0.0003/запрос
- Загрузить сценарий (вставить текст / PDF / картинка)
- Выбрать шаблон:

| Шаблон | Когда использовать |
|--------|--------------------|
| 📝 Свободный отчёт | Сценарий в виде скрипта или вопросов |
| 📊 Таблица оценки | Сценарий с чёткими пунктами |
| 🎭 Тональность | Важно КАК общался персонал |
| 🚨 Критические нарушения | Нужен быстрый вердикт |

- Нажать **Анализировать** (~10–40 сек в зависимости от провайдера)
- Уточнить детали в чате
- Скопировать отчёт

## Форматы сценария

| Формат | Как обрабатывается |
|--------|--------------------|
| PDF | PDF.js — прямо в браузере, без сервера |
| Картинка / скриншот | Groq Vision API (всегда, независимо от провайдера анализа) |
| TXT | Читается напрямую в браузере |

## Советы

- Каждый шаблон заканчивается разделом **"Проверяющему"** — что ты мог упустить при опросе
- Если AI неправильно понял сценарий — уточни в чате: *"пункт 3 означает..."*
- Можно комбинировать: сначала таблица, потом в чате попросить расширить конкретный пункт
- Groq ключ нужен всегда — даже если для анализа используется DeepSeek (Groq читает картинки)
